Contexto

¿Por qué un pipeline de datos cripto?

❌ Sin pipeline
  • Datos de precios dispersos y volátiles
  • Sin historial: cada consulta re-llama la API
  • Rate limit frágil — solo 10 000 req/mes gratis
  • Analizar datos = escribir código manualmente
✅ Con este pipeline
  • Ingesta batch automática cada hora y diario
  • Historial acumulado en BigQuery estructurado
  • Landing replayable — datos nunca se pierden
  • Preguntas en lenguaje natural, sin SQL
Objetivo: CoinGecko → BigQuery → agente Q&A, todo serverless en GCP, coste ~$0 al día.

Datos

Arquitectura de datos: Bronze → Silver

5 TABLAS RAW — coingecko_raw

  • coin_markets — top 250, precio, market cap, %24h  hourly
  • market_global — dominancia BTC/ETH, activos  hourly
  • trending — monedas más buscadas 24h  hourly
  • coin_ohlc — velas O/H/L/C top 50, 7d  daily
  • coin_metadata — categorías, supply, descripción  daily
Replay: si se pierde una tabla BQ, se recarga desde GCS sin volver a llamar CoinGecko.

4 VISTAS CURATED — coingecko_curated

latest_prices — snapshot más reciente por moneda
dim_coin — id, symbol, name, categories[]
ohlc_timeseries — velas diarias limpias
market_global_daily — totales macro por día
Dedup: ROW_NUMBER() OVER (PARTITION BY id ORDER BY ingestion_time DESC) = 1

Agente 2 · Detalle

ADK Agent Engine — 3 herramientas

🔵 Tool 1
BigQueryToolset

Historial completo en coingecko_curated. Solo lectura (WriteMode.BLOCKED). Rankings, OHLC, dominancia.

🟡 Tool 2
get_live_price()

Precio en tiempo real desde CoinGecko. Sin esperar el job hourly. Precio, market cap, %24h.

🟢 Tool 3
get_crypto_news()

Noticias recientes del mercado. CoinGecko /news. Título, fuente, descripción y fecha.

El agente elige autónomamente qué tool usar según la pregunta — puede combinar las tres en una sola respuesta.
